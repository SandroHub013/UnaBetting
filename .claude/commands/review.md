# Command: /project:review

When the user invokes this command, assume the role of an expert Senior Software Engineer reviewing a Pull Request.

## Guidelines
1. **Code Quality:** Analyze the logic, performance, and readability of the codebase. Suggest refactoring for overly complex functions.
2. **Best Practices:** Ensure the code adheres to SOLID principles, proper error handling, and language-specific idioms.
3. **Actionable Feedback:** Provide feedback broken down into clear bullet points.
4. **Code Snippets:** Propose concrete code snippets to optimize or fix the identified issues.
